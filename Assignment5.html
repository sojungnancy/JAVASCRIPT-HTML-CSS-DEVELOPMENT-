Part 1: XML Schema

Game moves in Chess may be specified as follows: 

<moves>
   <move>
      <piece>Rb</piece>
      <to>bin</to>
      <from>a6</from>
   </move>
   <move>
      <piece>Pw</piece>
      <from>b5</from>
      <to>a6</to>
   </move>
</moves>

Develop an XML schema that captures such moves. Your schema should be such that the above set of moves should conform to the schema.

The following list of rules should apply to the XML instances permitted by your schema.

  1. The element moves should contain at least one move, but can contain more.
  2. The element move should contain one each of the elements: piece, to, and from. These elements can be in any order.
  3. The element piece must contain a valid piece name: Kb, Qb, Rb, Bb, Nb, Pb, Kw, Qw, Rw, Bw, Nw, or Pw.
  4. The element from must be a valid board location. For example, a8 and f5 are valid board locations, while p3 and e9 are not.
  5. The element to must be a valid board location as in from. However, for the to element bin is also valid (since a piece can be moved to the bin).

Please name your schema file UPI-Schema1.xsd where UPI is your UPI.

Schema V2

We now want to revise the previous schema making the piece field an attribute. Since the piece is central to a move, this attribute is always required in any valid move. The valid values of the attribute piece still follow the valid values of the piece element used in the first schema.

The following XML instance conforms to this revised schema. 

<moves>
   <move piece="Rb">
      <to>bin</to>
      <from>a6</from>
   </move>
   <move piece="Pw">
      <from>b5</from>
      <to>a6</to>
   </move>
</moves>

Develop the revised schema, and name this schema file UPI-Schema2.xsd where UPI is your UPI.

All schemas need to be checked using the XML schema validator supplied to you in the course — online validators may not be robust. Those that do not pass the supplied validator will not attract any mark.

Ensure you form your own additional test cases and test both of your schemas thoroughly. Make sure you have test cases that test schema conformance as well as non-conformance.

All schemas are to be hand-crafted. Machine-generated schemas will not attract any mark.
